# إصلاح الإشعارات Push - دليل شامل

## المشكلة الأساسية

الإشعارات لا تصل للكابتن ولا للراكب في الرحلات بين المدن لأن **Expo Push Token لا يُسجَّل بشكل صحيح**.

في الـ logs تظهر هذه الأخطاء:
```
WARN [Push] Failed to get driver push token: [Error: No "projectId" found]
WARN [Push] Failed to get passenger push token: [Error: No "projectId" found]
```

### لماذا يحدث هذا؟

عندما يحاول التطبيق الحصول على Expo Push Token، يحتاج إلى معرّف فريد للمشروع يسمى **Project ID**. بدون هذا الـ ID:

1. ❌ التطبيق لا يستطيع الحصول على Push Token
2. ❌ السيرفر لا يجد توكن ليرسل إليه
3. ❌ الإشعارات لا تصل لأي طرف

---

## الحل - 4 خطوات فقط

### الخطوة 1: إنشاء حساب على Expo

إذا لم يكن لديك حساب:
1. اذهب إلى [expo.dev](https://expo.dev)
2. اضغط **Sign Up**
3. أنشئ حساب (يمكنك استخدام GitHub أو Google)

### الخطوة 2: إنشاء Expo Project

1. بعد تسجيل الدخول، اذهب إلى **Projects**
2. اضغط **Create a new project**
3. أعطه اسم: `mosul-ride-app` أو أي اسم تفضله
4. اختر **Managed Workflow** (الخيار الافتراضي)
5. اضغط **Create**

### الخطوة 3: نسخ Project ID

1. بعد إنشاء المشروع، ستظهر صفحة المشروع
2. ابحث عن **Project ID** (عادة يكون في الأعلى أو في Settings)
3. انسخه - يبدو هكذا:
   ```
   a1b2c3d4-e5f6-7890-abcd-ef1234567890
   ```

### الخطوة 4: أضف Project ID في app.config.ts

افتح ملف `app.config.ts` وأضف Project ID في `extra`:

```typescript
const config: ExpoConfig = {
  name: env.appName,
  slug: env.appSlug,
  version: "1.0.0",
  // ... باقي الإعدادات ...
  
  extra: {
    eas: {
      projectId: "a1b2c3d4-e5f6-7890-abcd-ef1234567890"  // ← ضع Project ID هنا
    }
  }
};
```

## استكشاف الأخطاء

### المشكلة: لا تزال تظهر رسالة "No projectId found"

**الحل:**
1. تأكد من أنك أضفت Project ID بشكل صحيح في `app.config.ts`
2. أعد تشغيل التطبيق (اضغط `r` في Metro)
3. تأكد من أن الـ ID صحيح (انسخه مرة أخرى من expo.dev)

### المشكلة: الإشعارات لا تصل على الويب

**هذا طبيعي!** الإشعارات Push تعمل فقط على الأجهزة الحقيقية:
- ✅ iOS (جهاز حقيقي)
- ✅ Android (جهاز حقيقي)
- ❌ الويب (غير مدعوم)
- ❌ المحاكي (غير مدعوم)

### المشكلة: الإشعارات تصل لكن بدون صوت

**الحل:**
1. تأكد من أن صوت الجهاز مفعّل
2. تأكد من أن الإشعارات مفعّلة في إعدادات الجهاز
3. على Android: تحقق من أن قناة `driver-alerts` و `passenger-alerts` مفعّلة
